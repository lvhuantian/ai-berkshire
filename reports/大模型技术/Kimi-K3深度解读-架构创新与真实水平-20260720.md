# 看懂 Kimi K3：2.8 万亿参数的开源前沿模型

> 写作日期：2026-07-20（发布后第 4 天）。本文所有关键数字经过多信源交叉核实，厂商自报且尚无法独立验证的数字均已单独标注。技术报告和权重预计 2026-07-27 前发布，届时部分结论需要更新（见文末"待解问题"）。

## 一句话总结

Kimi K3 是月之暗面 2026-07-16 发布的 2.8 万亿总参数稀疏 MoE 旗舰模型，靠混合线性注意力（KDA）+ 跨层残差检索（AttnRes）等一揽子架构改动把扩展效率做到 K2 的约 2.5 倍（官方口径），独立评测确认它站上了全球第三档——与 Claude Opus 4.8、GPT-5.5 同档，Agent 类任务是它最强的一面，但整体仍落后 Claude Fable 5 和 GPT-5.6 Sol。它同时承诺 7 月 27 日前开放全部权重，如果兑现，将是首个 3T 级别的开源模型。

## 二、架构创新：相对 K2 改了什么

理解 K3 的变化，要先锚定 K2 的基线（K2 有公开技术报告 arXiv:2507.20534 和开放权重，是全文最扎实的部分）：

**K2 基线：**1.04T 总参数 / 32B 激活，注意力用 MLA（沿袭 DeepSeek-V3 设计，但专家数 256→384、注意力头 128→64），优化器用 MuonClip（Muon + 权重衰减 + RMS 匹配 + QK-Clip，通过更新后重缩放 Wq/Wk 抑制注意力 logits 爆炸），在 15.5T token 上预训练且全程零 loss 尖峰。

K3 在此基础上做了五组改动。下面每一组都先补齐背景概念，再讲 K3 具体怎么做——不需要机器学习基础也能读。

### 1. Kimi Delta Attention（KDA）：MLA → 混合线性注意力

K3 最大的架构变化。要看懂它，先补三层背景：

**背景一：注意力为什么又慢又贵。**大模型生成每个新字时，都要回头"看一遍"前文所有内容，逐个判断哪些和当前最相关——这就是注意力机制，也是大模型能理解长文的根本。但它的代价随文本长度平方增长：文本长 10 倍，计算量涨 100 倍。而且推理时要把所有历史内容的中间结果（键值缓存）存在显存里随时备查，到 100 万 token 时，光这份缓存就大到离谱。这是长上下文"又慢又贵"的根源。K2 用的 MLA 是对缓存做压缩的改良版，但本质仍是"每个字都回看全部历史"的全注意力。

**背景二：线性注意力换了一种记忆方式。**它不再保留全部历史，而是把读过的内容边读边压缩进一个"固定大小的记忆状态"，新字来了只跟这个状态交互。好处立竿见影：计算量随长度线性增长（长 10 倍只贵 10 倍），显存占用恒定，不再随文本膨胀。坏处也很直接：压缩必有损失，"大海捞针"式的精确回忆（比如从 80 万字前的合同里抠出某个数字）会变差——这是线性注意力多年没能上位的原因。

**背景三：delta 规则让固定记忆不变糊。**普通线性注意力往记忆里只加不改，写多了就像一块反复叠便利贴的白板，越叠越糊。delta 规则的做法是每次写入前先看记忆里的旧内容，用新信息去修正旧信息——先擦掉过时的一小块，再写上新的。KDA 在此基础上做了更细的门控：记忆的每个通道各自决定遗忘多少、保留多少。可以理解为一块会主动做勘误的白板，而不是无限叠便利贴。

**K3 的做法是混合，不是全换：**全模型约 3/4 的层用 KDA（拿走线性注意力的省钱曲线），每 4 层保留 1 层带输出门控的 MLA 全注意力（保底精确检索能力，"针"还是找得到）。这条路线延续自月之暗面 2025 年下半年的 Kimi Linear 研究，K3 是它第一次被推上万亿级旗舰。官方称百万 token 上下文下解码速度最高提升 6.3 倍（**厂商自报，验证投票 2-1，是全文最弱的一个数字，**权重开放前无法复现）。

### 2. Attention Residuals（AttnRes）：跨层选择性检索

**背景：残差连接是深度网络的"信息高速公路"。**大模型内部是几十上百层的流水线，信息逐层加工。如果每层都把上一层的结果彻底重做一遍，信息传到深层早就面目全非，训练信号也传不回浅层——深度网络根本练不动。2015 年 ResNet 给出的解法沿用至今：每层的输出 = 上一层的结果 + 本层的增量修改。相当于每个工位只在半成品上做小改动，原件始终顺着传送带往下传。这就是残差连接，此后所有深度网络的标配。

**它的局限：只有一条传送带，没有回头路。**传统残差是固定的逐层累加——第 60 层拿到的是前 59 层所有修改叠加后的"总和"。如果它想要的是第 3 层的某个原始信息（比如词面本身的特征，而不是后面加工出的抽象语义），只能指望这个信息在层层叠加中没被冲淡，没有任何直接通道可以回去取。

**AttnRes 的做法：在层与层之间也做一次"注意力"。**让每一层可以按需回头，从之前任意一层选择性地检索它的加工结果——我现在最需要哪一层的中间产物，就直接去调哪一层，而不是被动接收传送带递来的总和。打个比方：传统残差像传话游戏里每人只能拿到前面所有人叠加过的笔记，AttnRes 允许直接翻任何一位前手的原始笔记。

**效果与边界：**这项工作有独立论文（arXiv:2603.15031），报告约 25% 训练效率提升——达到同样水平少花约 1/5 算力（官方表述是"匹配 1.25 倍算力基线的 loss"）。注意规模口径：这是在 48B/3B 的小模型、1.4T token 上测得的，不是 K3 实际训练运行的独立测量。

### 3. Stable LatentMoE + 分位数负载均衡

**背景：MoE 是"总参数巨大、每次只用一小撮"的分工结构。**把模型里最占参数的部分拆成许多"专家"，每个字进来时由一个路由器打分，挑出最相关的几个专家来处理。这样总参数可以做得极大（决定知识容量的上限），而每个字实际动用的计算只有一小部分（决定推理成本）。K3 把这个结构推得很激进：896 个专家每次只激活 16 个（1.8%）——所以它总参数 2.8 万亿，但每生成一个字只动用约 500 亿，推理成本跟一个 50B 模型同一量级。专家规模相对 K2 翻了一倍多（384 选 8 → 896 选 16）。

**难点：负载均衡，即防止专家"旱的旱死、涝的涝死"。**路由器天然会偏爱少数几个"红人专家"——它们被练得越多就越好用，越好用就越被选中，正反馈之下，热门专家大排长队（训练变慢、显卡利用率崩掉），冷门专家几乎接不到活（白占着参数没练出本事）。传统解法是在训练目标里加一项"辅助损失"来惩罚分配不均——但这是间接手段，相当于靠罚款引导，而且这项惩罚会和"把模型练好"的主目标打架，调不好就两头受损。

**K3 的分位数负载均衡（Quantile Balancing）：**不靠罚款事后纠偏，而是从路由分数的分位数直接推导专家分配——相当于按分数排位强制分流，从机制上保证各专家的负载均衡，训练目标里就不用再塞惩罚项了。（DeepSeek-V3 的"免辅助损失"策略是同一方向的先例，K3 用分位数给出了自己的实现。）至于名字里的 Latent（潜空间）具体指什么结构，官方博客没有展开，要等技术报告披露。

### 4. Per-Head Muon 优化器 + SiTU 激活函数

**背景：优化器决定每一步训练怎么改参数。**训练就是拿数据反复调整几万亿个参数，优化器负责决定每一步"往哪个方向调、调多大幅度"（行业默认是 AdamW）。月之暗面是 Muon 路线的旗手：Muon 把参数矩阵的每步更新做正交化处理，让更新量在各个方向上更均匀——不让某几个方向独吞步长——同样的数据能涨得更快。K2 用的 MuonClip 在此之上加了 QK-Clip，专治万亿规模训练中注意力数值爆炸导致的训练崩溃，成果是 15.5T token 全程零 loss 尖峰（这在万亿级训练里是很硬的工程成绩）。

**K3 的 Per-Head Muon：把优化粒度切到"每个注意力头"。**注意力内部由许多平行的"头"组成，每个头是一个独立的小注意力单元，各自盯着不同侧面（有的看语法搭配，有的看远距离指代）。此前 Muon 对整块参数矩阵一起做正交化，等于让所有头共享一套步伐；Per-Head Muon 改为逐头独立处理，各头按自己的情况调整，互不干扰。

**SiTU（Sigmoid Tanh Unit）：换了激活函数。**激活函数是每层计算后加的非线性变换，是神经网络能学会复杂规律的来源（没有它，一百层网络等价于一层）。SiTU 替代了近年主流的 SwiGLU 一类，设计动机官方未展开，细节待技术报告。

### 5. 从 SFT 阶段起的量化感知训练

**背景：量化就是用更粗的数字换更小的显存。**模型参数通常用 16 比特浮点数存，量化是改用更低精度（8 比特、4 比特）来存——显存占用和读取带宽立省数倍，代价是数字变粗糙、可能损失精度。传统做法是"训练后量化"：先用高精度练完，再整体压缩，像拍完高清照再压图，损失只能事后弥补。

**K3 的做法：从监督微调（SFT）阶段起就带着低精度约束训练。**模型在训练过程中就适应了"数字只能这么粗"的现实，把精度损失在训练中消化掉，而不是训练完再补救。具体格式是 MXFP4 权重 + MXFP8 激活（MX 是微缩浮点格式的行业标准，英伟达 Blackwell 一代硬件原生支持，算力吞吐远高于 16 比特）。

**为什么这一条对开源意义重大：**2.8 万亿参数如果按 16 比特存，光权重就要约 5.6TB 显存，几乎没人部署得起；MXFP4 直接砍到约四分之一（1.4TB 量级），且这份低精度权重是"原生"的、不是压缩残次品。可以说没有这一条，"3T 级模型开源给别人用"就只是个姿态。

**关于"2.5 倍扩展效率"的准确读法：**官方原文的归因是"结构性改动 + 精炼的训练与数据配方"共同作用，**不能**把 2.5 倍单独记在架构头上（这个错误归因在核实中被 0-3 否决）。且该数字为厂商自报，技术报告发布前无法独立验证。

## 三、真实水平：独立评测怎么说

这是对"讲懂 K3"最关键的部分，因为厂商基准和独立评测给出的画面并不完全一致。

### 独立评测（Artificial Analysis，2026-07-17）

- 智能指数 **57 分，总排名第 3**——与 Claude Opus 4.8、GPT-5.5 同档，落后 Claude Fable 5（59.86）和 GPT-5.6 Sol。
- **Agent 类基准是最强项：**GDPval-AA v2 Elo 1668（超过 GPT-5.5 的 1494 和 Opus 4.8 的 1600，低于 Fable 5 的 1750 和 GPT-5.6 Sol 的 1748）；AutomationBench-AA 以 53% 排第 1。
- **成本效率突出：**每任务实测成本 $0.94，约为 Opus 4.8（$1.80）的一半；输出 token 比 K2.6 少 21%（132M 对 166M），即"话变少了但更准了"。

### 社区盲测

Arena.ai 的 Frontend Code Arena（前端代码盲测）中以 1679 分排第 1，超过 Claude Fable 5 的 1631，较 K2.6 从第 18 名跃升 17 位（标注为初步结果，且仅此单一基准）。

### 厂商自报基准（未独立复现，谨慎采信）

月之暗面自己的定位很诚实：**承认整体落后 Fable 5 和 GPT-5.6 Sol，**但称编码与 Agent 基准上大体胜过 Opus 4.8 和 GPT-5.5。MarkTechPost 整理的 11 项基准中，K3 在 5 项领先（Program Bench、SWE Marathon、BrowseComp、Automation Bench、OmniDocBench），但在最难的推理和仓库级任务上落后：DeepSWE 67.5 对 GPT-5.6 Sol 的 73.0，FrontierSWE 81.2 对 Fable 5 的 86.6，HLE-Full 43.5 对 53.3。

**注意：**Fortune 报道的"与 Fable 5 表现相当"是过度拔高，已被证伪——正确表述是官方自认落后于 Fable 5，但差距在收窄。

### 综合判断

K3 的真实位置：**开源模型第 1、全球第三档、Agent 任务尤其能打、性价比是同档最强。**它没有追平最前沿的两家（Fable 5、GPT-5.6 Sol），但把"开源模型与闭源顶尖的差距"压到了历史最小。

## 四、定价与开源：便宜中国模型时代的落幕信号

- **纵向看是大幅涨价：**未缓存输入 $3/M，是 K2 发布价（$0.60）的 5 倍。The Decoder 的评论标题很点题——"超便宜中国 AI 时代的终结信号"。
- **横向看仍有性价比：**输出 $15/M，高于 GLM-5.2（$4.40）和 DeepSeek V4 Pro（$0.87），但只有 Opus 4.8（$25）的六成、Claude Fable 5（$50）的三成。缓存命中输入 $0.30/M 对 Agent 场景（大量重复上下文）很友好。
- **开源承诺：**2026-07-27 前开放完整权重，号称"全球首个 3T 参数级开源模型"（实际 2.8T，官方话术口径）。截至本文写作日权重尚未上 HuggingFace，"开源"仍处承诺阶段；许可条款（是否沿用 K2 的修改版 MIT 及商用归属要求）未披露。

## 五、给 Agent / 基模方向从业者的几点启示

1. **架构方向的信号：**混合线性注意力（3/4 线性 + 1/4 全注意力）在旗舰规模上被验证可行，这是继 MiniMax、Qwen 之后又一家头部厂商押注线性注意力路线，长上下文 Agent 的推理成本曲线会继续下移。
2. **稀疏度继续推向极限：**896 选 16（1.8%）的稀疏度比 K2（384 选 8，约 2.1%）更激进，"总参数狂飙、激活参数克制"仍是共识路径。
3. **量化感知训练前移到 SFT 阶段**是值得注意的工程决策——低精度不再是部署阶段的妥协，而是训练阶段的一等公民。
4. **Agent 基准与通用智能基准的分化**在 K3 身上非常明显：它在 GDPval、AutomationBench、前端代码上能赢 Opus 4.8，但在 HLE 这类最难推理任务上差距仍大。选型时"看什么榜"比"看总榜"更重要。
5. **缓存定价结构（命中打一折）**加上 1M 上下文，对多轮 Agent 工作流的实际成本影响比牌面单价更大，测算成本时要按自己的缓存命中率算。

## 六、信息可信度分级与待解问题

**最扎实：**K2 基线（有技术报告 + 开放权重）；AA 独立评测数字；官方定价页数字。

**较扎实但溯源单一：**K3 全部架构细节（16/896、KDA、AttnRes 等）——虽经多方转述交叉印证，但最终都来自月之暗面单方披露，要等权重开放后从 config.json 独立确认。

**厂商自报、待复现：**2.5 倍扩展效率、6.3 倍解码加速（全文最弱数字）、25% 训练效率（小模型上测得）、编码/Agent 胜过 Opus 4.8/GPT-5.5 的自报基准。

**已证伪、勿引用：**2.7T 参数量（Fortune）；"与 Fable 5 相当"（Fortune）；把 2.5 倍效率单独归因于架构。

**待技术报告解答的问题：**
- 预训练数据规模（token 数）、后训练/RL 方法、完整优化器配置（目前只知 Per-Head Muon 和 SFT 起步的量化感知训练，K2 的 15.5T token 不能外推）。
- 开源许可具体条款；2.8T 权重的实际部署门槛（显存需求、推理框架对 MXFP4 的支持）。
- 精确激活参数量（约 50B 为社区推算）。
